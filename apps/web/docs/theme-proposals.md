# UI 主题改造方案

> 目标：让 `LoadingPage` 和 `create-iframe` 与 `GameRoomShell` 形成统一的游戏视觉语言。
>
> 分析依据：`GameRoomShell` 的实际设计令牌并**非**紫色系，而是 **暖金 + 深暗** 系：
> - 主文字色：`#fff7d8`（暖白/羊皮纸）
> - 金色强调：`#d4b15c`（游戏内 `--color-gold`）
> - 边框：`rgba(224,216,185,0.22)` 暖米白
> - 背景底色：`#07041a` 极深蓝黑
> - 字重：`font-weight: 950` 极粗
> - 文字阴影：`0 2px 10px rgba(0,0,0,0.85)` 强烈暗影
>
> 而当前 LoadingPage / create-iframe 的主色是 `violet-500`（`#7c3aed`），与游戏内调性相差甚远。

---

## 方案 A — "暗金仪式" 🏆

**核心思路**：放弃现有的科技/星空紫色调，全面对齐游戏内的**暖金 + 深暗 + 重字**风格。  
用户在 Loading → 创建房间 → 进入游戏三个阶段看到一致的色彩语言，无割裂感。

---

### A-1 设计令牌替换

| 原值（LoadingPage / create-iframe） | 替换为 | 用途 |
|------------------------------------|--------|------|
| `violet-500`（`#7c3aed`） | `#d4b15c`（金色） | 主强调色 |
| `#c4b5fd`（紫文字） | `#fff7d8`（暖白/羊皮纸） | 主文字、图标色 |
| `text-slate-500` | `rgba(255,247,216,0.55)` | 次级文字 |
| `text-slate-400` | `rgba(255,247,216,0.38)` | 说明文字 |
| `violet-500/[0.12]`（按钮背景） | `rgba(212,177,92,0.12)` | 按钮/卡片填充 |
| `border-violet-500/30` | `rgba(207,176,91,0.28)` | 边框 |
| 紫色粒子 `rgba(196,181,253,...)` | 金色粒子 `rgba(212,177,92,...)` | 环境粒子 |
| `0 0 12px rgba(139,92,246,0.25)` | `0 0 16px rgba(212,177,92,0.22)` | 按钮光晕 |

背景渐变从紫蓝星空改为：

```css
/* 方案A 背景：向游戏内深暗底色靠拢，带极淡暖色 */
background: linear-gradient(
  160deg,
  #06040f 0%,
  #0a0812 40%,
  #0d0c14 70%,
  #07041a 100%
);
```

### A-2 LoadingPage 改造

**整体隐喻**：从"指纹识别/科技连接"改为"**月亮升起/游戏召唤**"。

#### 图标区域
```
原：Fingerprint（lucide），violet/30，扫描线
改：🌕 月亮 SVG 圆形（或使用游戏内角色徽章轮廓）
    - 底座：径向渐变，中心 rgba(212,177,92,0.15) → transparent
    - 图标边缘：金色细边框 border: 1px solid rgba(212,177,92,0.35)
    - 动效：金色光晕脉冲 (pulse-glow)，替换紫色扫描线
```

```css
@keyframes lp-pulse {
  0%, 100% { box-shadow: 0 0 18px rgba(212,177,92,0.18), 0 0 40px rgba(212,177,92,0.06); }
  50%       { box-shadow: 0 0 32px rgba(212,177,92,0.40), 0 0 64px rgba(212,177,92,0.16); }
}
```

#### 文字区域
```
原：RETRIEVING DATA...  （slate-500，tracking-[0.6em]）
改：SUMMONING PLAYERS   （rgba(255,247,216,0.55)，tracking-[0.4em]，font-weight:800）

原副文字：Synchronizing with the oracle（italic，slate-500/50）
改副文字：传唤仪式进行中               （rgba(255,247,216,0.30)，tracking-[0.18em]，不斜体）

#### 足部标识
```

#### 环境层（新增）
```

## 方案 B — "紫金双轨" ✨

**核心思路**：保留现有紫色星空美学（已有设计工作）作为"**进入游戏前的宇宙空间**"氛围，  
同时强力注入金色（`#d4b15c`）作为主强调色，构建"**紫色宇宙 → 金色游戏**"的视觉通道。  
改动量小于方案A，紫色只退居装饰层（粒子、光晕），金色主导所有**交互元素**。

---

### B-1 色彩层级调整

```
第一层（背景/氛围）：深紫蓝渐变  → 保留，不变
第二层（装饰粒子）：紫色粒子 50% + 金色粒子 50%  → 平衡比例
第三层（容器/卡片）：边框改为金色系  → rgba(207,176,91,0.22)
第四层（交互元素）：按钮、选中态、进度 全部金色  → #d4b15c 系
第五层（主文字）：从 slate-500 提亮至 rgba(255,247,216,0.72)  → 暖白
```

---

### B-2 LoadingPage 改造

**整体隐喻保留**：指纹/连接感保留，但加入游戏感装饰元素。

#### 图标区域
```
保留：Fingerprint 图标（形态不变）
升级：
  - 图标色：从 violet-500/30 改为 rgba(212,177,92,0.50)（金色半透明）
  - 外圈光晕：新增圆形边框 border: 1px solid rgba(207,176,91,0.20)
               padding: 20px，rounded-full
  - 扫描线：改为金色 rgba(212,177,92,0.60)，保留 lp-scan 动效

      每段竖线：background #d4b15c，动效同 seat-speaking-bar 节奏
      排列：水平，间距 3px，高度 12~4~12px 交替

## 两方案对比

| 维度 | 方案 A：暗金仪式 | 方案 B：紫金双轨 |
|------|----------------|----------------|
| 与 GameRoomShell 一致性 | ⭐⭐⭐⭐⭐ 完全对齐 | ⭐⭐⭐⭐ 高度接近 |
| 改动量 | 大（背景、图标、全部颜色） | 中（保留背景，替换交互色） |
| 现有工作保留比例 | ~30% | ~70% |
| 视觉层次感 | 深暗 + 暖金，厚重神秘 | 星空紫 + 游戏金，轻盈灵动 |
| 适合风格 | 沉浸式游戏入口，强仪式感 | 游戏+科技双重感，年轻化 |
| 过渡到游戏内 | 无缝（同色系） | 有轻微色调切换 |
| 风险 | 需重写较多样式 | 低，大部分是颜色替换 |

---

## 推荐

- 如果追求**极致沉浸感和品牌一致性** → **方案 A**  
- 如果希望**快速落地、保留现有代码** → **方案 B**  
- 两方案在技术层面均无需改动组件结构，只需修改颜色值和少量动效
